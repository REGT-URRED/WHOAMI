# Global Rules

- Responde de forma directa, técnica y concisa. Sin relleno ni emojis.
- Lee siempre un archivo antes de editarlo.
- Prefiere extender código existente sobre crear archivos nuevos.
- Verifica antes de entregar: build, lint y tests deben pasar.
- Nunca expongas secrets, API keys o credenciales en código, logs o respuestas.
- Ante ambigüedad, diferencia hechos observados de hipótesis.
- Usa llamadas paralelas cuando las operaciones sean independientes.
